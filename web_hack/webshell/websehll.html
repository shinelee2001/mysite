웹쉘 실행을 위한 전제 조건

  1. 웹 어플리케이션 서버에서 사용하는 언어로 제작된 웹쉘 사용 (Server-Side Script 확장자 파일)
  2. 웹 어플리케이션 서버의 웹 디렉터리 내 웹쉘 존재
    3. 즉, 서버의 파일 시스템에 웹쉘이 존재하여야 한다. 보통 이때 file-upload 취약점 공격을 한다.
  3. 보안 솔루션이 존재할 경우 우회
    4. 보안 솔루션 존재의 예시
    • Block before upload
      • IPS, WAF, ...
    • Block/deleted immediately after upload
      • Anti-virus software, Webshell detection solutions, ...

웹쉘 핵심 동작 원리

공격자가 client-side에서 시스템 명령어를 입력/또는 그러한 명령어를 실행하는 파일을 업로드하면 server-side에서 해당 입력값을 받고 해당 명령어/또는 파일을 실행한다.

핵심은 바로 시스템 명령어 사용함수를 실행시키는 것이다!
