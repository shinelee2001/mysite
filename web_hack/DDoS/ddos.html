DDoS 공격의 원리

모든 서버는 요청을 수신한 뒤 응답한다. DDoS 공격은 감당하지 못할 정도로 응답할 작업이 늘어나면 서버가 과부하를 일으킨다는 취약점을 이용한다. 해커는 우선 서로 연결된 디바이스 네트워크(컴퓨터나 노트북, 스마트 냉장고 등간의 네트워크)를 구성한다. 이를 봇넷을 구성한다고 한다. 악성코드를 감염시켜 다른 표적을 공격하는 데 사용할 준비를 마친다. 감염된 디바이스가 변함없이 작동하는 경우가 많아 소유자들이 감염 사실을 알기 어렵다. (보안 패치나 소프트웨어 업데이트를 자주 하자). 네트워크 관리자가 봇을 감지하기 쉽지 않다. 봇마다 IP주소가 다르며, 합법적으로 보이기 때문이다. 이후 문제가 발생하면 어디서 시작되었는지 추적도 쉽지않다.

세 가지 주요 DDoS 공격 유형

1. 애플리케이션 계층 공격

서버에서 사용자 요청 시 웹 페이지를 생성하는 부분에 공격을 집중한다. 봇에게 페이지를 반복해서 요청하도록 명령하여 서버에 과부하를 일으킨다.

애플리케이션 계층 공격 중 HTTP Flood 공격은 다음과 같이 이루어진다.

  • 특정 URL 호출
  • 웹 페이지에서 특정 이미지 요청
  • GET 요청을 통해 문서 요청
  • 잦은 페이지 새로 고침 요청

보통의 사용자도 사이트를 방문해서 동일한 요청을 수행하는 경우가 많아 악의적인 트래픽과 합법적인 트래픽을 구분하기가 매우 어렵다.

2. 프로토콜 공격

로드 밸런서와 같은 시스템 구성요소를 대상으로 대규모 요청을 보내 플러딩 공격을 실행한다.

대표적으로 SYN Flood 공격이 있다. 해커가 소스로 스푸핑된 IP 주소와 함께 다수의 초기 연결 요청(SYN 요청)을 서버에 전송한다. 서버는 다음과 같이 반응한다.

  • 연결 요청 수신
  • 연결 완료를 요청하는 응답 전송
  • 해당 응답 대기
  • 지나치게 많은 응답 대기로 인한 마비

3. 볼륨 공격

대역폭을 표적으로 플러딩 공격을 실행하여 사용 가능한 공간을 부족하게 만들고, 이로 인해 서버를 무력화하는 것이 목적이다. UDP Flood, ICMP Flood 등.

대표적으로 DNS 증폭(Amplification) 공격이있다. 해커는 봇에게 대량의 데이터를 차지하는 정보를 요청하도록 명령한다. 그 다음 데이터를 다시 본래 서버로 보내게 한다. 이러한 방식으로 매우 빠르게 대규모 혼란을 일으킬 수 있다.

대응 방법

  • 블랙홀 필터링: 제한기준을 정하여 이러한 기준에 미치지 못하는 트래픽을 삭제한다.
  • 스크러버: IT팀이나 호스팅 회사가 공격 직전에 트래픽이 어떻게 바뀌었는 지 검사하여 악성 주소를 식별한다. 스크러버는 이러한 주소가 연결되는 것을 차단한다.
  • 캐스팅: 트래픽을 서버 네트워크로 분산시킨다. 용량을 늘리면 트래픽 과부하로 인해 요청에 응답하지 못하는 상황을 방지할 수 있다,

예방 방법

  • 애플리케이션 프런트 엔드 하드웨어: 시스템으로 유입되는 패킷을 면밀히 조사하여 위협 수준에 따라 관리한다.
  • 애플리케이션 수준 KCI(Key Completion Indicator): 대량의 수신 트래픽이 합법적인지, 협공의 일부인지를 판별한다.
  • DDS: 문제를 조기에 식별하여 사용자의 개입 없이 대응하는 디바이스를 사용해 프로토콜 공격과 볼륨 공격을 해결한다.
  • 방화벽: 인터넷과 서버 사이에 보호 계층을 삽입한다. 맞춤형 규칙으로 요청을 필터링하거나, 혹은 단순한 규칙을 사용해 공격자에게서 수신되는 트래픽을 모두 차단한다.
  • 속도 제한: 사용자가 특정 시간 동안 서버에 연결할 수 있는 횟수를 정의한다.
