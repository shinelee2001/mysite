Types of DB

  1. Relational Database: RDB
    2. 데이터를 열과 행을 가진 테이블 형태로 표현하며, 구조적으로 데이터를 관리한다.
    관계형 데이터베이스를 관리하는 MS(management system) 종류:
    • MySQL
    • Oracle
    • SQLite
    • MariaDB
    • PostgresSQL
      • ...
  2. NoSQL (Non-relational Database)
    3. 비관계형 데이터베이스의 종류:
    • Key-value type (Redis)
    • Document type (MongoDB)
    • Wide-column store DB (Cassandra)
    • Graph DB (Neo4j)
      • ...

Summary:

SQL DB NoSQL DB
Table 구조 Document 구조
Schema 준수 DB를 유연하게 변경가능
관계 형성에 적합 복잡한 관계에 부적합
수직적 확장 수직적/수평적 확장

그 외 계층형, 네트워크형, 객체지향형 데이터베이스 등이 존재한다.

Structured Query Language(SQL)

  • DDL(Data Definition Language)
    • CREATE, DROP, ALTER ...
  • DML(Data Manipulation Language)
    • SELECT, INSERT, UPDATE, DELETE ...
  • DCL(Data Control Language)
    • CRANT, REVOKE ...

데이터 처리를 위한 CRUD

대부분의 컴퓨터 소프트웨어가 가지는 기본적인 데이터 처리 기능인 Create, Read, Update, Delete를 묶어서 일컫는 말로서, 사용자 인터페이스가 갖추어야 할 기능(정보 참조/검색/갱신)을 가리키는 용어로도 사용된다.

SQL문으로 대응시키면, C->UBSERT, R->SELECT, U->UPDATE, D->DELETE로 대응할 수 있다.

기본 문법

  1. CREATE, DROP
    • CREATE DATABASE [db_name]
    • CREATE TABLE [table_name] ([column_name1] [data_type1], [column_name2] [data_type2], ...)
    • DROP [DATABASE/TABLE] [name]
  2. INSERT, SELECT, UPDATE, DELETE
    • INSERT INTO [table_name] ([column1], [column2], ...) VALUES ([value1], [value2], ...);
    • SELECT [column1, ...] FROM [table] WHERE [condition];
    • UPDATE [table] SET [column1]=[value1] WHERE [condition];
    • DELETE FROM [table] WHERE [condition];
